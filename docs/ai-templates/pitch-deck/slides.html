02 / THE SOLUTION

Efficiency by Exception.
AI classifies every claim before HR opens a single case.

BUCKET A — PASSED AI REVIEW
High-confidence, zero-violation claims. HR fast-tracks with a full AI audit record attached. No manual policy lookup required.
⚠
BUCKET B — REQUIRES ATTENTION
Flagged violations, partial approvals, employee edits. AI surfaces the exact policy clause breached — HR arrives with evidence, not a stack of attachments.
🔍
THE FRAUD TRAP
Employee edits any AI-extracted field → system silently records deviation, assigns HIGH / MEDIUM / LOW risk. Permanent, immutable signal surfaced to HR before they open the claim.

"We don't replace HR. We give them a compliance analyst that processes every receipt against policy before they open a single case."

Document Agent
PARALLEL OCR · ≤4 WORKERS
›Images → Llama 3.2 Vision
›PDFs → PyMuPDF → GLM-5.1
›ThreadPoolExecutor, ≤4 calls
›Extracts merchant, amount, flags
10 receipts run concurrently. Total time = slowest receipt, not the sum.
⚖
Compliance Agent
4-NODE REACT PIPELINE
›load_context (user + policy + receipt)
›check_conditions (ReAct, ≤5 calls)
›build_judgment → APPROVE / FLAG
›save_reimbursement + line_items
Any pipeline failure → MANUAL REVIEW. No claim is ever silently dropped.

07 / UNDER THE HOOD — FRAUD TRAP MECHANICS

A permanent, immutable fraud signal baked into every submission.

HIGH
Amount multiplied significantly. Mandatory HR flag.
MED
Amount increased above AI extraction.
LOW
Minor field edits. Logged, not flagged.
NONE
No edits. Clean audit record.
CHANGE_SUMMARY JSONB — WRITTEN ON EDIT
{
  "has_changes": true,
  "overall_risk": "HIGH",
  "changes_by_field": {
    "total_amount": {
      "original":  85.00,
      "edited":   850.00,
      "delta_pct": 900
    }
  }
}
This record is permanent and immutable. HR sees the original AI value vs. what the employee submitted — field by field — before they open the claim.
